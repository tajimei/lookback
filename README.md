#  look back

> 「予定をこなすだけで終わらせない」をコンセプトに開発した、自己成長をサポートするWebアプリケーション

日々のスケジュール管理に「振り返り」と「可視化」を組み合わせることで、あなたの時間の使い方をより深く理解し、より良い明日へのアクションを促します。

---

##  コンセプト

「look back」は、単なるスケジュール管理ツールではありません。

- **予定を立てる** → **実行する** → **振り返る** → **改善する**

このサイクルを簡単に回せるようにデザインされています。

毎日の「理想」と「実際」の差を可視化し、週単位・月単位で自分の成長を追跡できます。

---

##  主な機能

###  **カレンダー・スケジュール管理**

- 月間カレンダーで直感的に予定を管理
- 各日のイベントを一目で確認
- 「星マーク」で重要な日を記録
- クリックでその日の詳細ビューに移行

###  **日単位ビュー（デイリービュー）**

最も詳細な「1日の記録」を残すビューです。

#### 理想のスケジュール
- 朝立てた理想のスケジュールを時系列で記録
- 時間帯別に活動を可視化

#### 実際の結果
- 実際に行った活動を記録
- To-Doリストで日中のタスク管理

#### 振り返り
- その日を通じて気づいたことや成果を自由に記入
- 自動保存で進捗を安全に記録

#### 出費管理
- 日々の支出を用途別に記録
- リアルタイムで1日の合計金額を表示

###  **週単位ビュー（ウィークリービュー）**

時間の使い方の全体像を把握するビューです。

#### 切り替え表示
- **予定モード**: 理想的にどう���間を使いたかったか
- **実績モード**: 実際にどう時間を使ったか

#### カテゴリ別スタックバーチャート
- 勉強、バイト、サークル、プライベートなど、カテゴリ別の活動時間を積み上げグラフで表示
- 複数日の比較が容易

#### 目標タグ機能
- カスタム目標を色分けして追加
- 目標に向かって実績を積み上げる様子を可視化

###  **月単位振り返り（マンスリーレビュー）**

1ヶ月間の活動を統計的に振り返るビューです。

#### 時間配分の円グラフ
- **「How I spent my time?」**: 全時間をカテゴリ別に分類
- 月間の時間配分が一目で理解できます

#### カテゴリ別詳細分析

**バイト**
- 時給を入力すると、月間勤務時間から自動的に収入を計算
- 給与管理に役立つデータを即座に確認

**勉強**
- 勉強内容別（例：英語、数学）の合計時間を表示
- 各科目にどれだけの時間を費やしたかが可視化されます

#### 出費サマリー
- 月間全体の支出を日付・用途・金額とともに表示
- 家計管理の参考資料に

#### 星付き振り返り一覧
- 「星マーク」をつけた特別な日の振り返りコメントを表示
- 月間の成功体験を思い出す

##  技術仕様

### 開発環境・言語
| 項目 | 内容 |
|------|------|
| **Frontend** | HTML5, CSS3, Vanilla JavaScript |
| **グラフライブラリ** | Chart.js (CDN経由) |
| **データ永続化** | LocalStorage API |
| **対応ブラウザ** | Chrome 60+, Firefox 55+, Safari 12+, Edge 79+ |

### アーキテクチャの特徴

#### サーバーレス設計
- すべてのデータをブラウザのLocalStorageに保存
- サーバー連携不要で、プライバシーが完全に保護されます
- インターネット接続なしでオフライン利用も可能

#### フレームワークレス実装
- React、Vue、Angular等の大型フレームワークは不使用
- Vanilla JavaScriptのみでDOM操作を実行
- ファイルサイズが小さく、読み込みが高速

#### 保守性の高いスタイル管理
- CSS変数を活用してテーマカラーを一元管理
  - `--primary-color`: プライマリーグリーン
  - `--shadow-sm/md/lg`: 統一されたシャドウ
- DRYの原則に基づいた効率的なCSS

##  使い方

1. このリポジトリをクローン、またはダウンロード
```bash
git clone https://github.com/tajimei/lookback.git
cd lookback
```

2. `index.html` をブラウザで開く
```bash
# ローカルサーバーを起動する場合（推奨）
python -m http.server 8000
# または
npx http-server
```

3. ブラウザで `http://localhost:8000` にアクセス

### 基本的な使用フロー

#### １ : スケジュールを作成（カレンダービュー）
- カレンダーで任意の日付をクリック
- 「日単位で見る」を選択して詳細ビューへ

#### ２　: 理想のスケジュールを立てる
- 朝の時点で、その日の「理想のスケジュール」を入力
- 開始時刻・終了時刻・活動名・カテゴリを指定

#### ３ : 実際の結果を記録
- 1日を過ごす中で、実際に行った活動を「実際の結果」に追加
- To-Doリストでタスク管理

#### ４ : その日の振り返りを記入
- 「振り返り」セクションで、気づきや成果をコメント
- 重要な日は星マークをつける

#### ５ : 支出を記録
- 日中の出費を逐次「出費」セクションに記録
- 用途と金額を入力すると自動集計

#### ６ : 週単位で傾向を確認
- 「週単位で見る」でその週の活動傾向をグラフで確認
- 目標設定して目標達成度を可視化

#### ７ : 月末に月間振り返り
- 「今月の振り返り」で円グラフと統計情報を確認
- 次月の改善点を検討

##  実装のコツ

### Vanilla JS での状態管理

複数ビューの表示制御を効率的に行うため、グローバル状態を最小限にしつつ、必要な情報はLocalStorageから動的に読み込む設計としています。

```javascript
function showView(view) {
  // 各ビューの表示/非表示を制御
  document.getElementById('calendar-view').style.display = 'none';
  document.getElementById('daily-view-panel').style.display = 'none';
  // ...
  document.getElementById(view).style.display = 'block';
}
```

### Chart.js の動的データバインディング

週単位・月単位ビューのグラフは、LocalStorageのデータを集計して動的に生成されます。

```javascript
function renderMonthlyPieChart() {
```
